# Plan: Changes Management — Blazor WASM Frontend

---
> **HARD REQUIREMENT — NO AUTHORED JAVASCRIPT**
> This repository must contain zero hand-written `.js` files. Machine-generated JS (NuGet/npm build output, `_content/` bundles) is acceptable. Any plan step that would create or modify an authored `.js` file is **invalid** and must be replaced with a C#/`IJSRuntime` equivalent. Blazor's `IJSRuntime` resolves dot-path identifiers natively (e.g. `"__TAURI__.core.invoke"`), eliminating the need for JS wrapper files.
---

## Context
Building the Blazor WASM frontend for the first vertical slice of the WSG Tauri desktop app: a GUI for managing git changes in WSL repos. The design system ("Orchestrated Terminal") is fully specified in `.stitch/DESIGN.md` and the wireframe in `.stitch/changes/`. The goal is a fully wired end-to-end feature: Blazor UI → Tauri command → `wsl git` → render result. Visual target: Fork's layout/UX with GitKraken's polish. The git graph is explicitly out of scope for this slice. See VS-01.md for the Rust/Tauri backend commands this frontend calls.

## Component Stack

| Need | Solution | Reason |
|------|----------|--------|
| Repo tabs | Hand-rolled Tailwind | Full context switch (Fork model) — no tab body, no library fits |
| 4× resizable splits | `Toolbelt.Blazor.SplitContainer` | 59KB, 247K downloads, .NET 10, dedicated splitter |
| Staged/Unstaged file trees | Custom virtualized tree + `<Virtualize>` | Handles multi-thousand node repos with bad .gitignores |
| Diff viewer | `BlazorMonaco` (StandaloneDiffEditor) | Full diff mode, custom themeable via C# API, no JS to write |
| Add Repo modal | Hand-rolled Tailwind overlay | Glassmorphism per design spec, simple state machine |
| Commit inputs + everything else | Hand-rolled Tailwind | Design system already fully specified |

### `frontend/Wsg/Services/AppState.cs` (NEW)
Singleton — drives the entire app context (Fork tab model):
```csharp
using Wsg.Models;

namespace Wsg.Services;

public class AppState
{
    public List<RepoContext> Repos { get; } = [];
    public RepoContext? ActiveRepo { get; private set; }
    public bool ShowAddRepoModal { get; set; }
    public event Action? OnChange;

    public void AddRepo(RepoContext repo) { Repos.Add(repo); SetActiveRepo(repo); }
    public void SetActiveRepo(RepoContext repo) { ActiveRepo = repo; NotifyChange(); }
    public void NotifyChange() => OnChange?.Invoke();
}

public class RepoContext
{
    public required string WslDistro { get; init; }  // e.g. "Ubuntu", "Ubuntu-22.04", or "" for Windows
    public required string WslPath { get; init; }    // Linux path e.g. /home/user/repos/myrepo
    public bool IsWindows => string.IsNullOrEmpty(WslDistro);
    public string Name => WslPath.Split('/').Last();
    public string? Branch { get; set; }
    public string? SelectedFile { get; set; }
    public bool SelectedFileIsStaged { get; set; }
    public string CommitSubject { get; set; } = "";
    public string CommitDescription { get; set; } = "";
    public GitStatusResult? LastStatus { get; set; } // preserved on tab switch
}
```
Per-repo state preserved on tab switch — selected file, commit message, and last known status restored instantly without re-fetch flash.

### `frontend/Wsg/Services/GitService.cs` (NEW)
Scoped — single `IJSRuntime` touch point for the whole app:
```csharp
using Microsoft.JSInterop;
using Wsg.Models;

namespace Wsg.Services;

public class GitService(IJSRuntime js)
{
    private Task<T> Invoke<T>(string cmd, object args) =>
        js.InvokeAsync<T>("__TAURI__.core.invoke", cmd, args).AsTask();
    // Blazor resolves dot-path identifiers natively — no authored JS wrapper needed.

    // Repo discovery
    public Task<List<string>> GetWslDistrosAsync()
    public Task<string?> PickFolderAsync(string? startPath)
    public Task<bool> ValidateGitRepoAsync(string distro, string repoPath)

    // Git operations — all commands pass both distro and path
    public Task<GitStatusResult> GetStatusAsync(string distro, string repoPath)
    public Task<FileDiffResult> GetDiffAsync(string distro, string repoPath, string path, bool staged)
    public Task StageFileAsync(string distro, string repoPath, string path)
    public Task UnstageFileAsync(string distro, string repoPath, string path)
    public Task CommitAsync(string distro, string repoPath, string subject, string desc, bool amend)
}
```

### `frontend/Wsg/Pages/Components/DiffPane.razor` (NEW)
- Header: filename + commit SHA badge + expand/split icons
- `<StandaloneDiffEditor @ref="_diffEditor">` fills remaining space
- Custom Monaco theme defined in `OnAfterRenderAsync` (see Monaco Theme section)
- Content fed from `GitService.GetDiffAsync` which returns `FileDiffResult(Original, Modified)`.
  `StandaloneDiffEditor` requires two separate `TextModel` instances — **not** a unified diff string.
  After receiving `FileDiffResult`, call `Global.CreateModel` twice (each with a unique URI key)
  and pass them via `SetModel`:
  ```csharp
  var originalModel = await Global.CreateModel(jsRuntime, result.Original, "text/plain", "diff://original");
  var modifiedModel = await Global.CreateModel(jsRuntime, result.Modified, "text/plain", "diff://modified");
  await _diffEditor.SetModel(new DiffEditorModel { Original = originalModel, Modified = modifiedModel });
  ```
  `Global.CreateModel` signature: `Task<TextModel> CreateModel(IJSRuntime jsRuntime, string value, string language, string uri)`.
  Note: in BlazorMonaco 3.4.0 `Global` static methods **do** take an `IJSRuntime` as their first argument —
  inject `IJSRuntime` into `DiffPane.razor` and pass it to every `Global.*` call.
  The `uri` parameter is a required unique key used internally by Monaco to identify the model.
  Pass distinct URIs (e.g. `"diff://original"` and `"diff://modified"`) so re-calling on file switch
  reuses the same model slot rather than leaking new models.
  `DiffEditorModel` and `Global.CreateModel` are in the `BlazorMonaco.Editor` namespace.
- Inject `@inject IJSRuntime jsRuntime` — required for all `Global.*` calls (`Global.CreateModel`, `Global.DefineTheme`)
- Shows placeholder when no file selected

## Monaco Theme
Defined once in `DiffPane.razor` `OnAfterRenderAsync`. Call the static method on `BlazorMonaco.Editor.Global`.
In BlazorMonaco 3.4.0, `Global` static methods **do** take an `IJSRuntime` as the first argument — inject
`@inject IJSRuntime jsRuntime` at the top of `DiffPane.razor` and pass it to every `Global.*` call:
```csharp
await BlazorMonaco.Editor.Global.DefineTheme(jsRuntime, "orchestrated-terminal", new StandaloneThemeData {
    Base = "vs-dark",
    Inherit = true,
    Rules = [],
    Colors = new Dictionary<string, string> {
        ["editor.background"] = "#060e20",
        ["editor.foreground"] = "#dee5ff",
        ["diffEditor.insertedTextBackground"] = "#05cb6f1a",
        ["diffEditor.removedTextBackground"] = "#9f05191a",
        ["editorLineNumber.foreground"] = "#a3aac4",
    }
});
```
`DiffPane.razor` must inject `IJSRuntime` for these calls — in BlazorMonaco 3.4.0, `Global` static methods take `IJSRuntime` as their first parameter (confirmed by the library README: `Global.SetTheme(jsRuntime, ...)`). `StandaloneThemeData` is in the `BlazorMonaco.Editor` namespace.
